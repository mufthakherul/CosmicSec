# Agent Workflow Rules

These project rules are mandatory for future implementation work.

1. If replacing existing code with new code, archive the previous version first under `Archives/Codes/`.
2. Mark a feature as complete only when backend, frontend, routes, database, and gateway wiring are in place.
3. If creating a new report/audit/summary document, archive older versions under `Archives/Docs/`.
4. Prefer advanced, production-lean implementations over bare stubs when feasible and safe.
5. Update checklist checkboxes only after implementation is complete and validated.
6. After completing a task: run tests, commit, and push.
7. If replacing existing code with new code, archive the previous version first under `Archives/Tests/`.
